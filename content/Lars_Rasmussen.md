**Lars Rasmussen**, <a id="cite-note-1" href="#cite-ref-1">[1]</a>  

a Danish chess player and programmer, endgame and early [endgame tablebase](Endgame_Tablebases "Endgame Tablebases") researcher 
with interests in maximals and [mutual zugzwang](Zugzwang#MutualZugzwang "Zugzwang") <a id="cite-note-2" href="#cite-ref-2">[2]</a> <a id="cite-note-3" href="#cite-ref-3">[3]</a>. 
Along with [David Forthoffer](David_Forthoffer "David Forthoffer") and primary author [Sito Dekker](Sito_Dekker "Sito Dekker"), he contributed to the automated database generation program [Retro](index.php?title=Retro&action=edit&redlink=1 "Retro (page does not exist)"), considering both [en passant](En_passant "En passant") and [castling](Castling "Castling") <a id="cite-note-4" href="#cite-ref-4">[4]</a>, 
which revealed five discrepancies in KRKB <a id="cite-note-5" href="#cite-ref-5">[5]</a> 
from the numbers published by [Jaap van den Herik](Jaap_van_den_Herik "Jaap van den Herik") and [Bob Herschberg](Bob_Herschberg "Bob Herschberg"), et al. <a id="cite-note-6" href="#cite-ref-6">[6]</a>. 
Lars Rasmussen wrote an [MS-DOS](MS-DOS "MS-DOS") routine for [John Roycroft](John_Roycroft "John Roycroft") to search text for the extended [GBR code](https://en.wikipedia.org/wiki/GBR_code), which, when located, was thoroughly tested for self-consistency <a id="cite-note-7" href="#cite-ref-7">[7]</a>.
He further developed the [ChessBase](ChessBase_(Database) "ChessBase (Database)") interface program to access the [Thompson's Databases](Thompson%27s_Databases "Thompson's Databases") <a id="cite-note-8" href="#cite-ref-8">[8]</a> <a id="cite-note-9" href="#cite-ref-9">[9]</a>, and is author of an engine solving [pawn endgames](Pawn_Endgame "Pawn Endgame") <a id="cite-note-10" href="#cite-ref-10">[10]</a> <a id="cite-note-11" href="#cite-ref-11">[11]</a> <a id="cite-note-12" href="#cite-ref-12">[12]</a>.
